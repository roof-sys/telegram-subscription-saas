[![Python](https://img.shields.io/badge/Python-3.9+-blue.svg)](https://www.python.org/)
[![FastAPI](https://img.shields.io/badge/FastAPI-0.109-green.svg)](https://fastapi.tiangolo.com/)

[![Docker](https://img.shields.io/badge/Docker-Ready-blue.svg)](https://www.docker.com/)

>  **Production-ready SaaS платформа для монетизации Telegram-каналов через подписки**

Комплексная система бота с REST API, которая автоматизирует продажу подписок, обработку платежей (карты, СБП, USDT) и контроль доступа к Telegram-каналам. Разработана для реального коммерческого использования с живыми пользователями и транзакциями.

> ⚠️ **Важно:** Это портфолио-версия проекта. Реальные токены, ID каналов, платёжные ключи и некоторые бизнес-специфичные функции заменены на примеры в целях безопасности. Полная production-версия с конфигурациями доступна по запросу для работодателей.

## 🌐 REST API

Проект включает полнофункциональный REST API, построенный на FastAPI для аналитики и управления.

### Доступные эндпоинты:

#### 📊 Статистика
```http
GET /api/stats
```
Возвращает общую статистику: количество пользователей, платежи, доход, активные подписки.

**Пример ответа:**
```json
{
  "users": {"total": 150},
  "payments": {"total": 45, "completed": 40, "pending": 5},
  "revenue": {"total": 125000.0, "currency": "RUB"},
  "subscriptions": {"active": 30}
}
```

#### 👥 Пользователи
```http
GET /api/users?limit=50&offset=0
GET /api/users/{user_id}
```
Список пользователей с пагинацией и детальная информация о конкретных пользователях.

#### 💳 Платежи
```http
GET /api/payments?status=completed&limit=50&offset=0
GET /api/payments/{payment_id}
```
Фильтрация платежей по статусу, детальная информация о транзакциях.

#### 🔗 Webhook
```http
POST /api/webhook/payment
```
Автоматическая обработка уведомлений от платёжных систем.

### Swagger документация

Полная интерактивная документация API доступна по адресу:
```
http://localhost:8000/docs
```

После запуска API вы можете:
- Просмотреть все эндпоинты
- Тестировать запросы прямо в браузере
- Изучить схемы данных
- Копировать примеры кода

### Запуск API

```bash
# Вместе с ботом через Docker
docker-compose up -d

# Отдельно для разработки
uvicorn src.api.main:app --reload --host 0.0.0.0 --port 8000
```

API будет доступен по адресу `http://localhost:8000`

##  Установка

### Требования
- Python 3.9+
- Docker & Docker Compose (рекомендуется)
- PostgreSQL (если запуск без Docker)

### Вариант 1: Docker Compose (рекомендуется)

```bash

cd telegram-subscription-saas

docker-compose logs -f bot
```

##  Архитектура

### Процесс обработки платежей
```
Пользователь выбирает тариф
    ↓
Выбирает способ оплаты (Карта/СБП/USDT)
    ↓
Платёж создаётся в PostgreSQL со статусом "pending"
    ↓
Пользователь завершает оплату во внешней системе
    ↓
Вариант A: Пользователь нажимает "Проверить оплату"
Вариант B: Платёжная система отправляет webhook
    ↓
Бот/API проверяет статус через API/блокчейн
    ↓
При успехе:
    - Обновить статус платежа на "completed" в PostgreSQL
    - Создать запись подписки
    - Сгенерировать одноразовую пригласительную ссылку
    - Отправить пользователю
```

### Процесс контроля доступа
```
Пользователь присоединяется к каналу
    ↓
Бот перехватывает через событие ChatMemberUpdated
    ↓
Проверяет активную подписку в PostgreSQL
    ↓
Валидирует пригласительную ссылку (одноразовая, не истёкшая)
    ↓
Если невалидна → временно забанить пользователя
Если валидна → отметить ссылку как использованную, разрешить доступ
```

### Фоновый планировщик
- Запускается каждый час
- Проверяет все подписки на истечение в PostgreSQL
- Автоматически отзывает истёкший доступ

## 🔒 Безопасность

- ✅ **Переменные окружения** - все секреты хранятся в `.env` (никогда не коммитятся)
- ✅ **Шифрованное хранилище** - чувствительные данные шифруются перед сохранением на арендованных серверах
- ✅ **Транзакции PostgreSQL** - ACID гарантии, защита от race conditions
- ✅ **Подписи платежей** - MD5 подписи для запросов к API эквайринга
- ✅ **Одноразовые пригласительные ссылки** - автоистекающие ссылки предотвращают несанкционированное распространение
- ✅ **Валидация ввода** - все пользовательские данные санитизированы
- ✅ **Обработка ошибок** - graceful failure с логированием, без утечки данных
- ✅ **Изоляция Docker** - контейнеризация для безопасного развёртывания
